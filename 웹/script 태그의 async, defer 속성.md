HTML script 태그에는 async, defer 속성이 있다.

async, defer 속성이 true로 설정된 경우, script를 백그라운드에서 다운로드하고 브라우저는 HTML 파싱을 계속하도록 만든다.

async 속성은 다운로드만 백그라운드로 진행되고, 스크립트가 실행되는 단계에서는 HTML 파싱을 중단한다.

반면 defer 속성은 백그라운드에서 다운로드가 끝나더라도 HTML 파싱이 완전히 끝난 시점에 스크립트를 실행한다.

즉, 페이지 로딩 단계에서 다소 일찍 스크립트를 실행해줄 필요가 있다면 async를 사용하고, 그게 아니라면 defer를 사용해서 FCP를 개선할 수 있다.
