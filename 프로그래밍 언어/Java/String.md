자바에서 문자열은 여러 방법으로 할당할 수 있고, 각각의 방법으로 할당했을 때 차이가 존재한다.

## String 클래스를 new로 인스턴스화 하는 경우
String 클래스를 new로 인스턴스화 해서 문자열을 사용할 수 있다.

```java
String str1 = new String("문자열");
String str2 = new String("문자열");
```

str1, str2가 서로 같은 문자열을 갖고 있더라도, 둘은 별개의 메모리를 할당받은 다른 변수이다.

```java
str1 == str2 // false
```

## 문자열 리터럴을 사용하는 경우
String 클래스를 이용하지 않고 큰 따옴표를 사용해서 문자열을 할당할 수 있다.

```java
String str1 = "문자열";
String str2 = "문자열";
```

위에서 String 클래스를 사용한 경우와는 다르게, 이렇게 할당하는 경우 둘이 같은 값을 가지고 있으면 레퍼런스도 같다.

```java
str1 == str2 // true
```

그 이유는, 자바의 메모리 영역에 string constant pool이라는게 존재하기 때문이다.

## string constant pool
자바에서 new로 생성된 인스턴스는 메모리의 heap 영역에 새롭게 저장되고, 변수는 해당 레퍼런스를 참조하게 된다.

즉, new로 String 클래스의 인스턴스를 생성했을 때, 모두 다른 레퍼런스를 참조하게 된다.

근데 new가 아니라 큰 따옴표로 생성한 문자열의 경우, 먼저 string constant pool에서 같은 문자열 값을 가진 String 객체를 탐색하고, 존재한다면 해당 객체의 레퍼런스를 리턴한다.

따라서 큰 따옴표를 사용해 문자열을 할당할 경우, 같은 문자열 값을 할당받은 것들 끼리는 같은 레퍼런스를 공유하게 되는 것이다.
