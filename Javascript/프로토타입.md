자바스크립트는 프로토타입 기반 언어라고 불린다. 모든 객체가 속성과 메소드를 상속받기 위한 프로토타입 객체를 가지기 때문이다. 프로토타입 객체는 새로운 객체 생성을 위한 템플릿이라고 볼 수 있다.

## 생성자 함수로 인스턴스 생성
생성자 함수는 객체의 인스턴스를 생성하는 함수이다.

일반적인 함수와 생성자 함수 간의 문법적 차이는 없다. 단, 아래와 같은 규칙을 따른다.

1. 대문자로 시작
2. new 연산자와 함께 실행

인스턴스에 name, age 속성을 정의하는 생성자 함수 Person을 만들어 보자.

```javascript
function Person(name, age) {
  this.name = name;
  this.age = age;
}
```

그리고 Person을 new 연산자와 함께 실행하면 인스턴스가 생성된다.

```javascript
const person = new Person("Hoseung", 20);
console.log(person.name); // Hoseung
console.log(person.age); // 20
```

## prototype 객체
생성자 함수에서는 아래와 같이 프로토타입 객체에 접근할 수 있다.

```javascript
Person.prototype
```

이 prototype 객체가 바로 상속할 메소드와 속성들을 정의해두는 객체이다. prototype 객체 내에 있는 것이 아닌 속성과 메소드들은 상속되지 않는다.

```javascript
Person.prototype.hello = () => console.log("hello");
Person.bye = () => console.log("bye");

const person = new Person("Hoseung", 123);
person.hello(); // hello
person.bye(); // TypeError: person.bye is not a function
```

Person의 prototype 객체에 hello 메소드를 정의하고, 그냥 Person의 속성으로 bye 메소드를 정의했다.

그리고 인스턴스를 만들었을 때, hello는 실행할 수 있지만 bye는 실행할 수 없는걸 확인할 수 있다.

## 프로토타입 체인
지금까지의 동작을 보면 마치 prototype 객체에 정의된 것들이 새로운 객체에 복사되는 것처럼 보일 수 있다.

하지만 그것은 잘못된 생각이고, 객체의 속성과 메소드는 프로토타입 체인을 따라 올라가며 순차적으로 찾게 된다.

예를 들어, 바로 위의 예제에서 hello 메소드를 찾는 과정은 아래와 같다.

1. person 객체에 hello 메소드가 있는지 확인
2. person 객체의 프로토타입 객체 (Person.prototype)에 hello 메소드가 있는지 확인
3. Person.prototype에 존재하므로 실행

만약 프로토타입 체인을 끝까지 타고 올라갔음에도 찾지 못한다면, 그 때 타입 에러가 출력되는 것이다. 이것이 우리가 메소드를 실행할 때 일어나는 내부 동작이다.

위 과정을 자바스크립트 코드로 표현해본다면 아래와 같다.

```javascript
// 1. person 객체에 hello 메소드가 있는지 확인
if (new Object(person).hasOwnProperty("hello")) {
  person.hello();
} else {
  // 2. person 객체의 프로토타입 객체 (Person.prototype)에 hello 메소드가 있는지 확인
  if (Object.getPrototypeOf(person).hasOwnProperty("hello")) {
    // 3. 있으므로 실행
    person.hello();
  }
}
```

## prototype 객체 수정
새로운 속성이나 메소드를 정의하고 싶은 경우, prototype 객체를 수정할 수 있다.

```javascript
const person1 = new Person("Hoseung", 20);

Person.prototype.printSomething = () => console.log("javascript");

const person2 = new Person("Alvin", 20);

person1.printSomething(); // javascript
person2.printSomething(); // javascript
```

person1이 생성된 후에 prototype 객체를 수정했지만, person1과 person2 모두 같은 출력 결과를 보여준다. 그 이유는 똑같은 프로토타입 객체를 모든 인스턴스가 공유하기 때문이다.
