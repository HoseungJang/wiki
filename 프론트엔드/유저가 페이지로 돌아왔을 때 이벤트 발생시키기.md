## 요구사항
유저가 페이지를 떠났다가 다시 돌아왔을 때, 알람을 띄워줘야 하는 요구사항이 있었다.

## window focus 이벤트
처음에는 [이 글](https://blog.outsider.ne.kr/654)을 보고 window에 focus 이벤트 리스너를 걸어서 유저가 돌아온걸 인식하도록 만들었다.

### focus 이벤트의 문제점
데스크탑에서는 페이지로 돌아오자마자 바로 focus 이벤트가 trigger되기 때문에 유저가 페이지를 보자마자 바로 알람이 뜬다.

하지만 모바일에서는 페이지로 돌아온 후 실제로 터치까지 해서 focus를 주어야 이벤트가 trigger되어서 즉각적인 반응을 할 수가 없었다.

## visibilitychange 이벤트
그래서 그냥 현재 탭이 보여지고 있는지 아닌지를 구분할 수 있는 API를 찾아보다가 [Page Visibility API](https://www.w3.org/TR/page-visibility/)를 찾았다.

document에는 visibilityState가 있고, "visible", "hidden" 중 하나의 값이 담겨있고, boolean 값이 담겨있는 document.hidden으로도 visible 여부를 확인할 수 있다.

그리고 visibilityState의 변화는 visibilitychange 이벤트로 확인할 수 있는데, [호환성](https://caniuse.com/?search=visibilitychange)이 나쁘지 않다.

특정 케이스에서 제대로 동작하지 않는 경우가 있는 것 같긴 하지만, 내 use case에서는 브라우저별로 아무 문제없이 잘 동작했기에 바로 적용할 수 있었다.

```typescript
const handler = () => {
  if (document.visibilityState === "visible") {
    /* ... */
  }
}
document.addEventListener("visibilitychange", handler, { once: true });
```
